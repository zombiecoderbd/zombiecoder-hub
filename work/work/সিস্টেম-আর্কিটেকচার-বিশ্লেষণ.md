## ১. সার্ভার সংযোগ আর্কিটেকচার

### প্রধান সার্ভারগুলি:

**১. Next.js মেইন সার্ভার (পোর্ট ৩০০০)**
- লোকেশন: `http://localhost:3000`
- ফাংশন: HTTP API এন্ডপয়েন্ট, ওয়েব UI, অথেনটিকেশন
- গুরুত্বপূর্ণ এন্ডপয়েন্ট:
  - `POST /api/agent/chat` - এজেন্ট চ্যাট রিকোয়েস্ট
  - `POST /api/agent/session` - সেশন তৈরি
  - `POST /api/auth/login` - লগইন
  - `GET /api/health` - হেলথ চেক

**২. MCP WebSocket সার্ভার (পোর্ট ৩০০৩)**
- লোকেশন: `http://localhost:3003`
- ফাংশন: রিয়েল-টাইম বাই-ডিরেকশনাল কমিউনিকেশন
- প্রযুক্তি: Socket.IO
- ফাইল: `scripts/mcp-ws.ts` → `src/lib/mcp/websocket.ts`

### সংযোগ ফ্লো:

```
ক্লায়েন্ট
    ↓
Next.js সার্ভার (3000) - HTTP API
    ↓
AgentExecutor → ProviderManager → AI প্রোভাইডার
    ↓
ডেটাবেস (SQLite/PostgreSQL)
    ↓
MCP WebSocket সার্ভার (3003) - রিয়েল-টাইম স্ট্রিমিং
    ↓
ক্লায়েন্ট (Socket.IO ক্লায়েন্ট)
```

## ২. প্রোভাইডার কল ফ্লো

### প্রোভাইডার ম্যানেজার ফলব্যাক চেইন:

**ফাইল:** `src/lib/agent/provider-manager.ts`

**ফলব্যাক অর্ডার:**
1. **Ollama** (লোকাল, ফ্রি) - প্রাথমিক
2. **OpenAI** (ক্লাউড, পেইড) - ফলব্যাক ১
3. **Gemini** (ক্লাউড, ফ্রি টিয়ার) - ফলব্যাক ২
4. **Manual** (ইউজার ইনপুট) - শেষ অপশন

### কল ফ্লো:

```
POST /api/agent/chat
    ↓
AgentExecutor.execute()
    ↓
ProviderManager.sendMessage()
    ↓
┌─────────────────────────────────────┐
│ ১. Ollama.sendMessage()             │
│    লোকেশন: http://localhost:11434 │
│    মডেল: llama3, mistral, ইত্যাদি │
└─────────────────────────────────────┘
    ↓ (ফেইল করলে)
┌─────────────────────────────────────┐
│ ২. OpenAI.sendMessage()             │
│    API: https://api.openai.com      │
│    মডেল: gpt-4, gpt-3.5-turbo      │
└─────────────────────────────────────┘
    ↓ (ফেইল করলে)
┌─────────────────────────────────────┐
│ ৩. Gemini.sendMessage()            │
│    API: https://generativelanguage. │
│         googleapis.com              │
│    মডেল: gemini-pro, gemini-1.5    │
└─────────────────────────────────────┘
    ↓ (সব ফেইল করলে)
ম্যানুয়াল এরর মেসেজ
```

### কোড লোকেশন:
- **ProviderManager:** `src/lib/agent/provider-manager.ts`
- **AgentExecutor:** `src/lib/agent/executor.ts`
- **Ollama Provider:** `src/lib/agent/providers/ollama.ts`
- **OpenAI Provider:** `src/lib/agent/providers/openai.ts`
- **Gemini Provider:** `src/lib/agent/providers/gemini.ts`

## ৩. এজেন্ট কল ফ্লো

### এজেন্ট এক্সিকিউশন প্রসেস:

**ফাইল:** `src/lib/agent/executor.ts`

```
ইউজার মেসেজ
    ↓
POST /api/agent/chat (JWT ভ্যালিডেশন)
    ↓
AgentExecutor.storeMessage('user', message)
    ↓
AgentExecutor.getHistory(5) - শেষ ৫ মেসেজ
    ↓
AgentExecutor.execute(history + new_message)
    ↓
ProviderManager.sendMessage()
    ↓
AI প্রোভাইডার রেসপন্স
    ↓
AgentExecutor.storeMessage('assistant', response)
    ↓
ডেটাবেসে সেভ
    ↓
HTTP রেসপন্স রিটার্ন
```

### স্ট্রিমিং মোড:
- `stream: true` প্যারামিটার সহ রিকোয়েস্ট
- `AgentExecutor.executeStream()` ব্যবহার
- SSE (Server-Sent Events) ফরম্যাটে রিয়েল-টাইম চাঙ্ক
- Content-Type: `application/x-ndjson`

**১. নেটওয়ার্ক ল্যাটেন্সি:**
- HTTP রিকোয়েস্ট ট্রিপ টাইম
- ক্লাউড প্রোভাইডারের সাথে কানেকশন

**২. AI মডেল ইনফেরেন্স টাইম:**
- **Ollama (লোকাল):** প্রথম লোডে ২-৫ সেকেন্ড (মডেল লোডিং)
- **OpenAI:** ০.৫-২ সেকেন্ড (ক্লাউড ইনফেরেন্স)

**৩. ফলব্যাক চেইন ট্রায়াল:**
- যদি Ollama ফেইল করে → OpenAI ট্রাই
- যদি OpenAI ফেইল করে → Gemini ট্রাই

**৪. ডেটাবেস অপারেশন:**

- AuditLog ক্রিয়েট

### টাইমিং ব্রেকডাউন:
```
API কল শুরু: 0ms
JWT ভ্যালিডেশন: 10-50ms
ডেটাবেস সেভ (user): 20-100ms
ProviderManager.execute(): 500-5000ms
├─ Ollama ট্রাই: 2000ms (timeout)
├─ OpenAI ট্রাই: 1000ms (success)
└─ রেসপন্স প্রসেসিং: 50ms
ডেটাবেস সেভ (assistant): 20-100ms
AuditLog: 20-50ms
মোট: ~3-5 সেকেন্ড
```

## ৫. MCP WebSocket সার্ভার টুল কল

### MCP সার্ভার ইভেন্ট হ্যান্ডলার:

**ফাইল:** `src/lib/mcp/websocket.ts`

**ইভেন্ট টাইপ:**

১. **join-session**
   - JWT টোকেন ভ্যালিডেশন
   - সেশনে জয়েন
   - ইউজার-সেশন ম্যাপিং

২. **send-message**
   - ইউজার মেসেজ সেভ
   - AgentExecutor.execute() কল
   - রেসপন্স স্ট্রিম টু ক্লায়েন্ট

৩. **call-tool**
   - ToolExecutor.executeTool() কল
   - গভর্নেন্স ভ্যালিডেশন
   - টুল রেজাল্ট স্ট্রিম

৪. **disconnect**
   - ক্লিনআপ সেশন কানেকশন

### ToolExecutor টুলস:

**ফাইল:** `src/lib/agent/tools.ts`

**উপলব্ধ টুলস:**

১. **read-file** (LOW risk)
   - ফাইল কন্টেন্ট রিড
   - ম্যাক্স সাইজ: ১ MB
   - কনফার্মেশন: প্রয়োজন নেই

২. **write-file** (HIGH risk)
   - ফাইল রাইট/ওভাররাইট
   - কনফার্মেশন: প্রয়োজন
   - গভর্নেন্স চেক: WRITE operation

৩. **execute-command** (HIGH risk)
   - শেল কমান্ড এক্সিকিউট
   - টাইমআউট: ১৫ সেকেন্ড
   - কনফার্মেশন: প্রয়োজন
   - গভর্নেন্স চেক: EXECUTE operation

৪. **search-files** (LOW risk)
   - ফাইল প্যাটার্ন সার্চ
   - ম্যাক্স ডেপথ: ৬
   - ম্যাক্স রেজাল্ট: ২০০

### গভর্নেন্স ভ্যালিডেশন:

**ফাইল:** `src/lib/agent/governance.ts`

- ইউজার পারমিশন চেক
- রিস্ক লেভেল ভ্যালিডেশন
- পলিসি ভায়োলেশন চেক
- অ্যাপ্রুভাল রিকোয়ারমেন্ট

### MCP সার্ভার কি AI কল করে?

**না, সরাসরি AI কল করে না।**

MCP WebSocket সার্ভার:
- ক্লায়েন্ট থেকে মেসেজ রিসিভ করে
- AgentExecutor কে ডেলিগেট করে
- AgentExecutor → ProviderManager → AI প্রোভাইডার
- রেসপন্স স্ট্রিম করে ক্লায়েন্টে

টুল এক্সিকিউশন:
- ফাইল সিস্টেম অপারেশন (read/write)
- শেল কমান্ড এক্সিকিউশন
- ফাইল সার্চ

## ৬. WebSocket এডিটর কানেকশন সমস্যা

### সমস্যা বিশ্লেষণ:

**পর্যবেক্ষিত:** Socket.IO এডিটর কানেকশন হচ্ছে না

**সম্ভাব্য কারণসমূহ:**

১. **CORS রেস্ট্রিকশন**
   - কোড: `src/lib/mcp/websocket.ts` লাইন ৩১-৩৪
   ```typescript
   cors: {
     origin: process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000',
     credentials: true
   }
   ```
   - এডিটর যদি অন্য অরিজিন থেকে কানেক্ট করতে চায় → ব্লক হবে

২. **JWT টোকেন ভ্যালিডেশন**
   - `join-session` ইভেন্টে টোকেন প্রয়োজন
   - টোকেন মিসিং বা ইনভ্যালিড → ডিসকানেক্ট

৩. **পোর্ট কনফ্লিক্ট**
   - MCP WS সার্ভার পোর্ট ৩০০৩
   - যদি অন্য সার্ভিস এই পোর্ট ব্যবহার করে

৪. **সার্ভার রানিং না**
   - `npm run mcp:ws` রান করা না থাকলে
   - টেস্ট: `curl http://localhost:3003`

### সমাধান:

**১. CORS ফিক্স:**
```typescript
cors: {
  origin: '*', // ডেভেলপমেন্টের জন্য
  credentials: true
}
```

**২. টোকেন চেক:**
- এডিটর কনফিগে JWT টোকেন পাস করতে হবে
- টোকেন জেনারেট: `/api/auth/login` এন্ডপয়েন্ট

**৩. সার্ভার স্টাটাস চেক:**
```bash
curl http://localhost:3003
# আউটপুট: {"ok":true,"service":"mcp-ws","port":3003}
```

**৪. MCP WS ইনফো এন্ডপয়েন্ট:**
```
GET /api/mcp/ws-info
রেসপন্স: {"ok":true,"MCP_WS_URL":"http://localhost:3003"}
```
