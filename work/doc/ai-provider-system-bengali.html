📖 ভূমিকা - সহজ ভাষায় বুঝুন

ZombieCoder Hub একটি শক্তিশালী AI সিস্টেম যা বিভিন্ন AI প্রোভাইডার (যেমন Ollama, Gemini, OpenAI) ব্যবহার করে আপনার প্রশ্নের উত্তর দেয়। এই ডকুমেন্টে আমরা খুব সহজ ভাষায় বুঝবো কিভাবে আপনার ইনপুট AI প্রোভাইডারের কাছে যায় এবং উত্তর ফিরে আসে।

💡 মূল কথা: আপনি যখন চ্যাটবক্সে কিছু লেখেন, তখন সেটি কয়েকটি ধাপ পার হয়ে AI প্রোভাইডারের কাছে যায়, প্রোভাইডার উত্তর তৈরি করে, এবং সেই উত্তর আবার আপনার কাছে ফিরে আসে।

🔄 সম্পূর্ণ প্রবাহ - ইনপুট থেকে আউটপুট পর্যন্ত

ধাপে ধাপে কি ঘটে:

1 আপনার ইনপুট (User Input)
আপনি চ্যাটবক্সে লিখলেন: "Python এ factorial function লিখে দাও"
2 অথেন্টিকেশন চেক (Authentication)
সিস্টেম চেক করে আপনি লগইন করেছেন কিনা। JWT টোকেন দিয়ে আপনার পরিচয় যাচাই হয়।
3 সেশন তৈরি বা খুঁজে বের করা (Session Management)
প্রতিটি চ্যাট সেশনের একটি уникаль ID থাকে। আপনার সব মেসেজ সেই সেশনে সেভ হয়।
4 মেসেজ ডাটাবেজে সংরক্ষণ (Store Message)
আপনার প্রশ্ন MySQL ডাটাবেজের messages টেবিলে সেভ হয়।
5 এজেন্ট এক্সিকিউটর (Agent Executor)
এটি হলো মূল ব্রেইন! এটি ঠিক করে কোন AI প্রোভাইডারকে কল করতে হবে।
6 প্রোভাইডার ম্যানেজার (Provider Manager)
এটি অটোমেটিকভাবে প্রোভাইডার বেছে নেয়: Ollama → OpenAI → Gemini
7 AI প্রোভাইডারে কল (API Call)
আপনার মেসেজ নির্বাচিত প্রোভাইডারের API তে পাঠানো হয়।
8 উত্তর পাওয়া (Receive Response)
AI প্রোভাইডার উত্তর তৈরি করে ফেরত পাঠায় (৩০-৪৫ সেকেন্ড সময় লাগতে পারে)।
9 উত্তর সংরক্ষণ (Store Response)
AI এর উত্তর আবার ডাটাবেজে সেভ হয় ভবিষ্যতের জন্য।
10 আপনার কাছে উত্তর (Display Response)
উত্তর আপনার স্ক্রিনে দেখানো হয়!

📁 কোড ফাইলগুলো কোথায় আছে

১. প্রোভাইডার ফাইল (যারা API কল করে):

  • src/lib/agent/providers/ollama.ts - Ollama কে কল করে
  • src/lib/agent/providers/gemini.ts - Gemini কে কল করে
  • src/lib/agent/providers/openai.ts - OpenAI কে কল করে

২. ম্যানেজার ফাইল (যারা সিদ্ধান্ত নেয়):

  • src/lib/agent/provider-manager.ts - কোন প্রোভাইডার ব্যবহার করবে তা ঠিক করে
  • src/lib/agent/executor.ts - সম্পূর্ণ প্রক্রিয়া পরিচালনা করে

৩. API রাউট (যারা আপনার রিকোয়েস্ট পায়):

  • app/api/agent/chat/route.ts - চ্যাট মেসেজ হ্যান্ডেল করে
  • app/api/agent/session/route.ts - সেশন তৈরি/মুছে ফেলে

৪. ডাটাবেজ (যেখানে সব সংরক্ষিত হয়):

  • MySQL ডাটাবেজ: zombiecoder
  • টেবিল: messages, sessions, users, agents, tools ইত্যাদি

💾 ডাটাবেজে কি কি সংরক্ষিত হয়

টেবিলের নাম কি সংরক্ষিত হয় কেন লাগে
users ইউজারের তথ্য (নাম, ইমেইল, পাসওয়ার্ড) লগইন করার জন্য
sessions চ্যাট সেশনের তথ্য প্রতিটি চ্যাট আলাদা রাখতে
messages আপনার প্রশ্ন + AI এর উত্তর চ্যাট হিস্ট্রি দেখতে
agents AI এজেন্টের কনফিগারেশন এজেন্টের ব্যক্তিত্ব ঠিক রাখতে
tools AI কি কি কাজ করতে পারে ফাইল পড়া/লেখা, কমান্ড চালানো
audit_logs কোন AI কল কখন হয়েছে নিরাপত্তা এবং মনিটরিং
system_config সিস্টেম সেটিংস (মডেল লিস্ট) কনফিগারেশন সংরক্ষণ
📊 উদাহরণ: আপনি যখন "Hello" লেখেন, তখন messages টেবিলে ২টি এন্ট্রি তৈরি হয়:
  1. role: "user", content: "Hello"
  2. role: "assistant", content: "Hi! How can I help you?"

💻 কোড উদাহরণ - কিভাবে API কল হয়

Ollama কে কিভাবে কল করা হয়:

// Ollama প্রোভাইডার থেকে কোড (ollama.ts) async sendMessage(messages) { // আপনার কম্পিউটারে Ollama সার্ভারে কল const response = await fetch( 'http://localhost:11434/api/chat', // ← Ollama URL { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ model: 'smollm:latest', // ← কোন মডেল ব্যবহার হবে messages: [ { role: 'user', content: 'আপনার প্রশ্ন' } ], stream: false }) } ); return response.message.content; // ← উত্তর ফেরত }

Gemini কে কিভাবে কল করা হয়:

// Gemini প্রোভাইডার থেকে কোড (gemini.ts) async sendMessage(messages) { // Google এর সার্ভারে কল const response = await fetch( 'https://generativelanguage.googleapis.com/v1beta/' + 'models/gemini-flash-latest:generateContent' + '?key=AIzaSyB1hFbncerF7ZZ1sLz9NIOvFCnw6wmdMdM', // ← API key { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ contents: messages, generationConfig: { temperature: 0.7, // ← ক্রিয়েটিভিটি লেভেল maxOutputTokens: 2048 // ← সর্বোচ্চ উত্তর দৈর্ঘ্য } }) } ); return response.candidates[0].content.parts[0].text; // ← উত্তর }

ফলব্যাক লজিক (provider-manager.ts):

// ProviderManager - অটোমেটিক ফলব্যাক সিস্টেম async sendMessage(messages) { // ধাপ ১: Ollama চেষ্টা করো try { if (ollama available) { return ollama.sendMessage(messages); // ✅ সফল! } } catch { // Ollama ফেইল করলে... } // ধাপ ২: OpenAI চেষ্টা করো try { if (openai configured) { return openai.sendMessage(messages); // ✅ সফল! } } catch { // OpenAI ফেইল করলে... } // ধাপ ৩: Gemini চেষ্টা করো try { if (gemini configured) { return gemini.sendMessage(messages); // ✅ বর্তমানে এটি কাজ করছে! } } catch { // Gemini ফেইল করলে... } // সব ফেইল করলে return "দুঃখিত, কোন AI পাওয়া যাচ্ছে না!"; }

📊 বর্তমান সিস্টেম অবস্থা

✅ যা কাজ করছে:

  • Gemini AI: সম্পূর্ণ কাজ করছে (৫০+ মডেল)
  • ডাটাবেজ: MySQL সফলভাবে সংযুক্ত
  • অথেন্টিকেশন: JWT লগইন কাজ করছে
  • মেসেজ স্টোরেজ: প্রশ্ন-উত্তর সেভ হচ্ছে
  • ফলব্যাক সিস্টেম:自动ভাবে Gemini ব্যবহার হচ্ছে

⚠️ যা উন্নত করা যায়:

  • Ollama: টাইমআউট হচ্ছে (৩০ সেকেন্ডে) - টাইমআউট বাড়ানো যায়
  • OpenAI: API key কনফিগার করলে আরো ভালো হবে
  • রেসপন্স টাইম: ৩৫-৪৫ সেকেন্ড লাগছে - caching যোগ করা যায়

🎯 সারসংক্ষেপ

সহজ ভাষায় পুরো প্রক্রিয়া:

  1. আপনি লিখলেন: চ্যাটবক্সে আপনার প্রশ্ন
  2. সিস্টেম চেক করল: আপনি লগইন করেছেন কিনা
  3. সেশন খুলল: আপনার চ্যাটের জন্য একটি unique ID
  4. প্রশ্ন সেভ করল: MySQL ডাটাবেজে
  5. AI বেছে নিল: Ollama চেষ্টা → ফেইল → Gemini সিলেক্ট
  6. API কল করল: Google এর সার্ভারে আপনার প্রশ্ন পাঠাল
  7. উত্তর পেল: ৩৫-৪৫ সেকেন্ড পর Gemini উত্তর দিল
  8. উত্তর সেভ করল: আবার ডাটাবেজে
  9. দেখাল: আপনার স্ক্রিনে উত্তর!
🔑 মূল বিষয়: ZombieCoder Hub একটি স্মার্ট সিস্টেম যা automatically সবচেয়ে ভালো AI প্রোভাইডার বেছে নেয়। যদি একটি ফেইল করে,自动ভাবে অন্যটি ব্যবহার করে। বর্তমানে Gemini দিয়ে সব কাজ চলছে এবং এটি সফলভাবে উত্তর দিচ্ছে!

🚀 পরবর্তী পদক্ষেপ (Recommendations)

১. Ollama ঠিক করা:

  • টাইমআউট ৩০ সেকেন্ড থেকে ৬০ সেকেন্ডে বাড়ান
  • ছোট মডেল ব্যবহার করুন (যেমন: smollm:latest)
  • RAM এবং CPU চেক করুন

২. OpenAI যোগ করা:

  • OpenAI API key নিন (platform.openai.com)
  • .env.local এ OPENAI_API_KEY=sk-... যোগ করুন
  • আরো দ্রুত উত্তর পাবেন!

৩. পারফরম্যান্স উন্নত করা:

  • Streaming response যোগ করুন (উত্তর ধীরে ধীরে দেখাবে)
  • Caching সিস্টেম যোগ করুন (একই প্রশ্নের দ্রুত উত্তর)
  • Session management উন্নত করুন
