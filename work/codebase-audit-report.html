WebSocket সংযোগ বিশ্লেষণ

WebSocket কিভাবে যুক্ত আছে

Next.js App (Port 3000) | HTTP API Routes /api/agent/chat ---> AgentExecutor ---> ProviderManager /api/agent/session | | | | Ollama/OpenAI/Gemini | | [পৃথক প্রসেস] | | | scripts/mcp-ws.ts (Port 3003) | | | http.createServer() | | | MCPWebSocketServer (Socket.IO) | "join-session" | "send-message" | "call-tool" | AgentCore (core.ts) <--- ডেমো/Echo Response
সমস্যা: WebSocket সার্ভার এবং Next.js মূল অ্যাপ সম্পূর্ণ পৃথক প্রসেসে চলে। WebSocket সার্ভার (mcp-ws.ts) Port 3003-এ একটি আলাদা HTTP সার্ভার তৈরি করে, যেটি Next.js-এর runtime-এর সঙ্গে কোনো সংযোগ রাখে না।

WebSocket-এর বর্তমান সমস্যা তালিকা

1. দুই ভিন্ন Agent System ব্যবহৃত হচ্ছে

WebSocket সার্ভার src/lib/mcp/websocket.ts ব্যবহার করে AgentCore (core.ts), যেটি একটি echo/placeholder response দেয়:

// core.ts - Line 210-225 (generateResponse method) // This is a placeholder - real implementation would call LLM const response: AgentResponse = { sessionId: '', message: `I received your message: "${userMessage}". I am ${agent.name}, ready to help.`, confidence: 0.85, timestamp: new Date() };

অথচ HTTP API Route app/api/agent/chat/route.ts ব্যবহার করে AgentExecutor (executor.ts), যেটি আসল LLM Provider (Ollama/OpenAI/Gemini) কল করে।

ফলাফল: WebSocket দিয়ে মেসেজ পাঠালে শুধু echo পাওয়া যাবে, কোনো AI response আসবে না। HTTP API দিয়ে পাঠালে আসল AI response আসবে।

2. WebSocket Next.js-এর সঙ্গে সমন্বিত নয়

Next.js 14/15/16-তে custom WebSocket সার্ভার সরাসরি attach করা যায় না যেহেতু Vercel/Next.js নিজে HTTP সার্ভার manage করে। scripts/mcp-ws.ts একটি আলাদা http.createServer() তৈরি করে Port 3003-এ।

ইন্ডাস্ট্রি বেস্ট প্র্যাকটিস: Next.js-এ WebSocket লাগলে হয় একটি আলাদা WebSocket সার্ভার (যেমন ws/Socket.IO) সত্যিই আলাদা সার্ভিস হিসেবে চালাতে হবে (যা ঠিক আছে), অথবা Vercel-এর মতো platform ব্যবহার করলে Server-Sent Events (SSE) ব্যবহার করতে হবে। বর্তমান কোডে Port 3003 সার্ভার চালু করতে npm run mcp:ws কমান্ড দিতে হয়, কিন্তু এটি সেটআপ ডকুমেন্টেশনে পরিষ্কার বলা নেই এবং কোনো process manager (PM2, systemd) কনফিগার করা নেই।

3. WebSocket-এ Auth Validation এর ত্রুটি

websocket.ts Line 242-250: validateToken() method JWT verify করে কিন্তু token payload থেকে userId extract করে session-এর userId-এর সঙ্গে মিলিয়ে দেখে না। যেকোনো valid JWT token দিয়ে যেকোনো session-এ join করা সম্ভব।

private async validateToken(token: string): Promise<boolean> { try { const { verifyJWT } = await import('../auth/service'); const payload = await verifyJWT(token); return !!payload; // শুধু token valid কিনা চেক করে, userId match করে না } catch { return false; } }

পুরো codebase-এ কোথাও WebSocket client-side implementation নেই। কোনো React component Socket.IO-তে connect করে না। শুধু PowerShell test script আছে।

BUG-1: Duplicate Singleton (AgentCore) CRITICAL
